Game Logic

Here is a list of the logic flow

  1. Player Selects a Valid Piece (done)
  2. Player Selects a new location for said piece (done)
  3. Move is validated based on team/piece (done)
  4. Perform a capture if necessary (done)
  5. Move the actual piece (done)

The core of the game logic is in my move validator function which works based on 3 boolean values:
1. Is the final destination going to be a capture?
2. Is the move legal based on the type of piece being moved?
3. Is the path clear for the pieces that need it to be clear?

At the end of the validator function I test for those values and then act based upon the results, for instance different code is executed based on wheather the capture value is true or not.

I went to the internet to try and find some ideas on how to implement my logic but honestly there really isn't much out there that would integrate with the way built my web app so I am happy to say that this is all 100% my code/ideas.

Special Considerations

Every piece save for a Knight required a clear path for all moves

I nested my capture code at the tail end of the validator function that changes object properties as needed.

I added html5 audio element that is triggered to play when a team tries to make an invalid move, it's from the movie jurrasic park and I think it fits well.

I use html5 local storage as well to capture the user and timestamp for log in users.

I also use html5 local storage to save a current game and then load it again, you can either load the sample JSON or the user generated JSON (stored in localStorage).
